Ao se iniciar os estudos do comportamento e aplicação de cada linguagem, se torna nítido suas particularidades,
cada uma trazendo suas facilidades e dificuldades nas aplicações.

Começando pela Python, é uma linguagem extremamente legível e que com um conhecimento mínimo de inglês já é possível
entender a maioria das funcionalidades. Parte do que contribui para essa legibilidade é a ausência da necessidade de 
declarar explicitamente os tipos das variáveis, ou seja elas possuem tipos implicitos e dinâmicos, variando sua 
tipificação pelo conteúdo atribuído à Variável. Essa legibilidade é poderosa ao ponto de até mesmo com um código levemente
extenso como o de cálculo das média para as disciplinas, a visualização se torna simples, porém vem com um custo. Como 
o Python é uma linguagem interpretada, seu tempo de execução é maior, já que antes de começar a rodar, ele passa por um
sistema intermediário instalado no computador para processar as tarefas linha a linha progressivamente.
Apesar das facilidades, Python se torna uma linguagem menos flexível, já que as questôes implícitas e dinânimicas ficam
fora do controle do programador, principalmente nas questões de acesso a memória. Apesar de ser mais legível, a separação
de tarefas/blocos de código se dá pela identação, o que pode ser benéfico para reduzir a quantidade de caracteres no 
arquivo fonte, pode se tornar confuso em um código mais extenso.

Partindo para a C, já é perceptível algumas diferenças em relação ao Python, principalmente em relação a Tipagem das
variáveis e funções que, dessa vez, são necessariamente explicitas e de tipo estático, ou seja, não pode mudar sua 
formatação ao decorrer do código, apenas os valores. Outra mudança é que apesar da identação estar presente, ela não
necessária, pois o que separa os blocos de código são as chaves ({}), o que deixa mais simples identificar os inícios e
fins de cada bloco e/ou funções. A linguagem C é uma linguagem com uma extensa biblioteca de funções e é compilada, o que
torna o processo de transformação da linguagem de alto nível para um programa equivalente em código máquina, sem a
necessidade de instalar algum programa intermediário para trnasformar e executar o arquivo fonte. A linguagem C, apesar
de possuir uma tipagem estática e explicita, ela é uma linguagem extremamente versátil e que dá uma liberdade maior
em relação ao acesso à memória, criação de ponteiros para memória e certos controles do sistema operacional (Windows).

Já o JAVA é uma linguagem com uma legibilidade mais baixa, com suas declarações, semelhantes ao C, sendo explícitas e com
tipagem estática, mas em número que já deixa bem claro com o que a linguagem se especializa em trabalhar, enfatizando
classes, acessos públicos ou privados e as amarrações de funções. Para quem conhece alguns Paradigmas, fica claro que
o JAVA é próprio para trabalhar com orientação a objeto, tanto pela excessividade de declarações, quanto pelas próprias
funções básicas da linguagem como o "print", que é "system.out.println()",sendo uma junção de classes, sendo uma a classe
System que dá acesso às classes de entrada e saída, logo a "out" que instância a classe PrintStream, que possibilita chamar a 
classe "println" que escreve mensagens no terminal. No geral, tirando os aspectos de orientação a objetos e as funções 
com nomes gigantescos, ambas são bem parecidas em todos os aspectos, sendo as principais diferenças o C ainda ser mais 
versátil nas questões de alocação de memória que o JAVA e que o JAVA, assim como o Python, é uma linguaguem interpretada
e precisa de um sistema intermediário para processar o arquivo fonte.
